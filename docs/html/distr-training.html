Distributed trainingÂ¶

OpenSeq2Seq supports two modes for distributed training: simple multi-tower approach and Horovod-based approach. To run the mutli-GPU training using the first approach, you only need to change the configuration parameter num_gpus. To use Horovod you will need to set use_horovod: True in the config and start run.py script using mpirun. With Horovod you can also enable multi-node execution.

Note

num_gpus parameter will be ignored when use_horovod is set to True. In that case the number of GPUs to use is specified in the command line with mpirun arguments.
